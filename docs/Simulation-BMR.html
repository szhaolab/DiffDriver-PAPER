Description

Six signatures are generated from LUAD mutation data, using fasttopics model. The third signature is selected to generate the binary phenotype which has the maximum correlation with the third signature. The 63th nttype turns out to reach the maixmum of the third signture(signature plots). We then mannully place two hotspots within the the 63th annotation files. We demonstrate that even though phenotype have zero correlation with selection, all the benchmark methods have high type II error, i.e, clains pheotype correlates with selection. In contrast, diffdriver algorithm has low type II error.
